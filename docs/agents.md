# AI Agent Usage

## Tools Used

**Claude Sonnet (Anthropic)** — primary AI coding assistant, used via Claude Code CLI.

## What the AI Generated

### Scaffold & boilerplate
- Initial Vite + React project structure suggestion
- Component file naming and folder layout
- CSS reset and dark theme color palette suggestion

### SVG math
- `unrotatePoint` / `rotatePoint` utility functions (2D rotation matrix)
- Resize-with-rotation algorithm: unrotate both mouse points around shape center, compute local delta, apply per handle
- Rotation handle angle computation (`atan2`)

### Initial component drafts
- First drafts of `ShapeRenderer`, `SelectionHandles`, `Canvas`, `PropertiesPanel`, `Toolbar`
- `useShapes` hook structure and CRUD operations

## What Was Verified Manually

Everything generated by AI was reviewed and tested by hand before shipping:

| Area | What was checked |
|------|------------------|
| Resize logic | Traced through `dx/dy` for each of the 8 handles at 0° and 45° rotation to confirm correct edge anchoring |
| Rotation math | Verified angle wraps cleanly (mod 360, always positive) |
| Event propagation | Confirmed `stopPropagation` on shape `mousedown` prevents canvas deselect; confirmed shapes don't consume events when in non-select tool mode |
| `window` listeners | Verified `useEffect` cleanup removes listeners on unmount; verified `handleMouseMove` is stable via `useCallback([onUpdate])` |
| `dragRef` closure | Confirmed all shape data is captured into `dragRef` on `mousedown`; `mousemove` handler only needs `onUpdate` in its dep array |
| Text hit target | Confirmed invisible `<rect>` covers the text bounding box for reliable clicking |
| Initial shapes | Manually verified 5 default shapes display and are interactable on first load |
| Keyboard shortcuts | Tested all mapped keys; confirmed Input fields suppress shortcuts |

## What Was Changed From AI Output

- Replaced per-element `onMouseMove`/`onMouseUp` with `window`-level listeners via `useEffect` — the AI initially put handlers on the SVG element, which broke drag when the mouse left the SVG.
- Added the invisible hit-rect for text shapes — the AI draft only rendered the hit-rect when the shape was selected.
- Fixed stroke toggle: AI initially used a text input set to `'none'`; replaced with a checkbox to enable/disable stroke cleanly.
- Adjusted the `unrotatePoint` call in resize to correctly pass the **original** center (captured at `mousedown`), not the re-derived center from the mutating shape.
- Added `display: 'block'` to the SVG element to eliminate a 4 px bottom gap caused by inline SVG baseline alignment.

## Judgment Calls (Not Delegated to AI)

- **Scope**: Decided not to implement undo/redo, zoom/pan, or export. These are table-stakes for a real tool but beyond the stated "micro" scope and the time budget.
- **SVG vs Canvas API**: Chose SVG for declarative integration with React; Canvas API would require manual hit-testing and a separate render loop.
- **No external state library**: `useShapes` with `useState` is sufficient; adding Zustand/Redux would be premature.
- **Drift documentation**: Chose to document the resize-drift-at-steep-rotation limitation rather than implement a more complex iterative solver.
